## 🔒 Безопасность изменений

### ✅ Совместимость с продакшеном:

1. **SSL для PostgreSQL** - автоматически определяется:
   - Локальная разработка (localhost/127.0.0.1): `sslmode=disable`
   - Продакшен (Render/Railway): `sslmode=require`

2. **Новое поле `estimated_ready_at`**:
   - `nullable=True` - не ломает существующие записи
   - Старые заказы будут иметь `NULL` в этом поле
   - Новые заказы автоматически получат расчетное время

3. **Обратная совместимость API**:
   - Если расчет времени не удался, заказ все равно создается
   - Фронтенд проверяет наличие `result.estimated_ready` перед отображением

## 📊 Логика работы системы

### Расчет времени готовности:

1. **Базовое время**: 10 минут на заказ (настраивается)
2. **Активные заказы**: учитываются заказы со статусами "новый" и "в работе"
3. **Формула**:
   ```
   Время готовности = Σ(оставшееся время активных заказов) + 10 минут
   ```

### Пример:

- Заказ #1 создан 3 минуты назад → осталось 7 минут
- Заказ #2 создан 8 минут назад → осталось 2 минуты
- **Новый заказ #3** будет готов через: 7 + 2 + 10 = **19 минут**
